<Title>Popover positioning</Title>

`JBSelect` renders its option list in a popover. On desktop, the default `popoverPosition="absolute"` leaves that popover in the select's normal layout context. This is usually the right choice for ordinary page layouts.

Use `popoverPosition="fixed"` when that layout context would clip or incorrectly position the open list—for example, when the select is inside a scrollable panel (`overflow: auto`), a clipped container (`overflow: hidden`), or a nested positioned layout.

In fixed mode, the popover is anchored to the select box using viewport coordinates. It follows the select when its scrollable parent, the window, or the select's size changes.

> Fixed positioning is a desktop behavior. The component continues to use its mobile select UI at mobile breakpoints.

The panel below has `overflow-y: auto`. Scroll it until the selects are near an edge, then open each one. The first uses the default absolute positioning; the second uses `popoverPosition="fixed"`, which anchors its option list to the visible select instead of the panel's scroll area.

Use the prop directly on the React component:

```tsx
import { JBOptionList, JBSelect } from 'jb-select/react';

export function ProjectOwnerSelect({ people }) {
  return (
    <div style={{ height: '14rem', overflowY: 'auto' }}>
      <JBSelect
        label="Project owner"
        placeholder="Choose a person"
        popoverPosition="fixed"
      >
        <JBOptionList
          optionList={people}
          getTitle={(person) => `${person.name} ${person.family}`}
          getValue={(person) => person.userId}
        />
      </JBSelect>
    </div>
  );
}
```

## Positioned or clipped parents

An ancestor with `position: relative` is common in cards, drawers, and overlays. If that ancestor also clips overflow, an absolutely positioned option list can be cut off or use an unintended containing block. Fixed mode bases placement on the select's viewport rectangle instead.

## Choosing a position mode

<table>
  <thead>
    <tr>
      <th>Situation</th>
      <th>Recommended value</th>
      <th>Why</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Standard page, form, or non-clipping card</td>
      <td><code>"absolute"</code> (default)</td>
      <td>Keeps the popover in the normal component layout context.</td>
    </tr>
    <tr>
      <td>Select in an <code>overflow: auto</code> or <code>overflow: hidden</code> ancestor</td>
      <td><code>"fixed"</code></td>
      <td>Prevents the ancestor's overflow from clipping the desktop option list.</td>
    </tr>
    <tr>
      <td>Select nested in another positioned UI, such as a drawer or overlay</td>
      <td><code>"fixed"</code></td>
      <td>Anchors to the select's viewport position rather than an ancestor's positioning context.</td>
    </tr>
    <tr>
      <td>Mobile viewport</td>
      <td>Either value</td>
      <td>The component uses its mobile UI; this setting affects the desktop popover.</td>
    </tr>
  </tbody>
</table>

## Web-component usage

For the standalone custom element, set the property when creating the component in JavaScript:

```js
const select = document.querySelector('jb-select');
select.popoverPosition = 'fixed';
```

`popoverPosition` accepts `"absolute"` and `"fixed"`. Choose `fixed` only for containment or positioning problems; the default remains a good fit for the majority of layouts.
