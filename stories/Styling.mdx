<Title>Styling</Title>

`JBSelect` styling is split between the select host, the popover shell, and the option rows. Use `--jb-select-*` variables for the control, `--jb-option-*` variables for rows, and official parts only when a specific internal surface needs fine styling.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Select recipes should use those common tokens where possible, then map them into `--jb-select-*`, `--jb-option-*`, and `--jb-popover-*` variables where the select needs component-specific values.

## Styling Model

Use four layers when styling select examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-select/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-select/stories/styles).
- Public component variables use `--jb-select-*` on `jb-select`, `--jb-option-*` on `jb-option`, and `--jb-popover-*` on `jb-select::part(popover)`.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and one select-specific recipe from [`modules/jb-select/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-select/stories/styles). The style story applies both classes directly to `JBSelect`.

## Recommended Customization Path

1. Use `--jb-select-*` variables first for width, height, border, radius, value text, labels, helper text, and list spacing.
2. Use `--jb-option-*` variables for option row colors, active state, padding, radius, and row height. See [Options](./Options.mdx) for the option API.
3. Style the dropdown shell by setting `--jb-popover-*` variables on `jb-select::part(popover)`.
4. Use official parts such as `selected-value`, `search-input`, and `arrow-icon` only when a public variable is not specific enough.
5. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
6. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
7. Add a new select variable or part only when a needed internal surface cannot be customized with current public variables or official parts.

## Starter Recipe

Use this minimal recipe when a product only needs brand alignment across the select and dropdown.

```css
jb-select.brand-select {
  --jb-select-border-color-focus: #2563eb;
  --jb-select-border-color-selected: #2563eb;
  --jb-select-value-color: #0f172a;
  --jb-select-placeholder-color: #64748b;
  --jb-select-rounded: 0.75rem;
  --jb-option-bg-color-active: #dbeafe;
  --jb-option-color-active: #1d4ed8;
  --jb-option-border-radius: 0.5rem;
}

jb-select.brand-select::part(popover) {
  --jb-popover-bg-color: #ffffff;
  --jb-popover-padding: 0.375rem;
  box-shadow: 0 1rem 2rem rgb(15 23 42 / 0.14);
}

jb-select.brand-select::part(arrow-icon) {
  color: #2563eb;
}
```

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), then each component should keep its own style CSS in its story [`styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-select/stories/styles) folder.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-select/stories/styles/style-carbon.css */
jb-select.theme-name-select {
  /* select host and jb-option variables */
}

jb-select.theme-name-select::part(popover) {
  /* jb-popover background, radius, padding, and shell treatment */
}

jb-select.theme-name-select::part(selected-value) {
  /* selected value/chip styling when variables are not enough */
}
```

This keeps common theme tokens reusable while making the `JBSelect`, `JBOption`, and `JBPopover` styling APIs visible in one select page.
