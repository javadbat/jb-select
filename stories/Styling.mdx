## Styling component parts

`JBSelect` exposes the `selected-value`, `search-input`, and `arrow-icon` parts. Style them from outside the component with `::part(...)`.

<JBSelect
  className="part-styled-select"
  label="Selected number"
  value="7"
>
  <JBOptionList optionList={numberOptionList} />
</JBSelect>

```tsx
<JBSelect
  className="part-styled-select"
  label="Selected number"
  value="7"
>
  <JBOptionList optionList={numberOptionList} />
</JBSelect>
```

```css
.part-styled-select::part(selected-value) {
  border-radius: 62.4375rem;
  background: #dbeafe;
  color: #1d4ed8;
  font-weight: 700;
  padding: 0.125rem 0.5rem;
}

.part-styled-select::part(search-input) {
  color: #1d4ed8;
  caret-color: #1d4ed8;
}

.part-styled-select::part(arrow-icon) {
  color: #1d4ed8;
}
```

## Styling component states

`JBSelect` exposes the `open`, `disabled`, and `invalid` custom states. Use `:state(...)` to change variables only while that state is active. Open the first example to see its active state; the second shows the disabled state.

<JBSelect className="state-styled-select" label="Open this select" required>
  <JBOptionList optionList={numberOptionList} />
</JBSelect>

<JBSelect className="state-styled-select" label="Disabled select" value="7" disabled>
  <JBOptionList optionList={numberOptionList} />
</JBSelect>

```css
.state-styled-select:state(open) {
  --jb-select-border-color: #2563eb;
  --jb-select-border-color-focus: #2563eb;
  --jb-select-bgcolor: #eff6ff;
}

.state-styled-select:state(disabled) {
  --jb-select-bg-color-disabled: #f1f5f9;
  --jb-select-value-color-disabled: #64748b;
}

.state-styled-select:state(invalid) {
  --jb-select-border-color: #dc2626;
  --jb-select-message-color-error: #dc2626;
}
```

The `invalid` state becomes active when the component reports a validation error, for example after `reportValidity()`.
