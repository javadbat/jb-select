# Multiple selection

Use `JBSelect` with `multiple` when people need to choose more than one value from the same list—for example, assigning team members, selecting filters, or choosing permissions. The open list stays available while people add or remove options, and the selected value is an array.

## Features

- Select and deselect several options without reopening the list.
- Search the available options by typing after the select is focused.
- Render options with any JSX content, including `JBCheckbox`.
- Control the selected values with React state, or set the component value directly.
- Use strings, numbers, objects, or other JavaScript values as option values.
- Keep the standard `JBSelect` capabilities: label, helper message, placeholder, disabled state, clear button, validation, mobile UI, RTL, and sizing.

## Quick start

Add `multiple` and render a `JBOption` for every selectable value. In React, import the components from `jb-select/react`.

```tsx
import { JBOption, JBSelect } from 'jb-select/react';

export function AssigneeSelect() {
  return (
    <JBSelect
      multiple
      label="Assignees"
      placeholder="Choose one or more people"
    >
      <JBOption value="ava">Ava Patel</JBOption>
      <JBOption value="sam">Sam Lee</JBOption>
      <JBOption value="morgan">Morgan Jones</JBOption>
    </JBSelect>
  );
}
```

### What changes in multiple mode?

`value` is an array of the selected option values. Selecting an item adds it; selecting it again removes it. The displayed selected values are separated by commas, and the list remains open so people can continue choosing.

The component filters options as the user types. With the list open, use the arrow keys to move through options and <kbd>Enter</kbd> to select or deselect the active option.

## Controlled values

Pass an array to `value` and update that array from `onChange` when the selection changes. This is the recommended pattern when the chosen values are part of form or application state.

```tsx
import { useState } from 'react';
import { JBOption, JBSelect } from 'jb-select/react';

const people = [
  { id: 1, name: 'Ava Patel' },
  { id: 2, name: 'Sam Lee' },
  { id: 3, name: 'Morgan Jones' },
];

export function ControlledAssigneeSelect() {
  const [value, setValue] = useState<number[]>([1, 3]);

  return (
    <JBSelect
      multiple
      value={value}
      onChange={(event) => setValue(event.target.value)}
    >
      {people.map((person) => (
        <JBOption key={person.id} value={person.id}>
          {person.name}
        </JBOption>
      ))}
    </JBSelect>
  );
}
```

## Checkbox options

Checkboxes are optional, but they make the multi-select nature of a long list immediately obvious. Put a `JBCheckbox` inside each `JBOption`; `JBSelect` keeps it synchronized when an option is selected or deselected.

## HTML web-component usage

For the standalone web component, import `jb-select` once and use the same `multiple` attribute. Read or assign the array through the DOM property, not an HTML attribute.

```html
<jb-select id="assignees" multiple label="Assignees">
  <jb-option value="ava">Ava Patel</jb-option>
  <jb-option value="sam">Sam Lee</jb-option>
  <jb-option value="morgan">Morgan Jones</jb-option>
</jb-select>
```

```js
const assignees = document.querySelector('#assignees');

assignees.value = ['ava', 'morgan'];
assignees.addEventListener('change', (event) => {
  console.log(event.target.value); // ['ava', 'sam']
});
```

## Practical guidance

- Add a clear `label` and a `placeholder` that sets the expectation, such as “Choose one or more tags”.
- Use `JBOptionList` instead of manually mapping `JBOption` when an array-driven list does not need custom JSX.
- Use `popoverPosition="fixed"` when the select lives in a scrolling or overflow container.
- For external form validation, pass the message through `error`; use `required` for the built-in required rule.
