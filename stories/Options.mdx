# Supplying options

`JBSelect` accepts options in two ways: `JBOption` and `JBOptionList`. Both are placed inside `JBSelect`, both participate in search and keyboard navigation, and both can be used with single or multiple selection. The right choice depends on whether you need custom option markup or simply need to display an array.

## `JBOption`: render each option yourself

Use `JBOption` when the option needs custom content—such as a checkbox, icon, colour swatch, avatar, or secondary text. Set the value with `value` and place the visible JSX between the tags.

```tsx
import { JBOption, JBSelect } from 'jb-select/react';

const people = [
  { id: 1, name: 'Ava Patel', role: 'Designer' },
  { id: 2, name: 'Sam Lee', role: 'Engineer' },
];

<JBSelect label="Assignee">
  {people.map((person) => (
    <JBOption key={person.id} value={person.id}>
      <strong>{person.name}</strong>
      <span>{person.role}</span>
    </JBOption>
  ))}
</JBSelect>;
```

## `JBOptionList`: create options from an array

Use `JBOptionList` when the option data is already an array and each row can use a consistent layout. It creates the underlying `JBOption` elements for you.

### Arrays of primitive values

For strings or numbers, pass `optionList` directly. Each value is used for both the displayed title and selected value.

```tsx
import { JBOptionList, JBSelect } from 'jb-select/react';

const languages = ['TypeScript', 'JavaScript', 'Rust'];

<JBSelect label="Language">
  <JBOptionList optionList={languages} />
</JBSelect>;
```

### Arrays of objects

For objects, provide `getTitle` to choose the display text and `getValue` to choose the value returned by `JBSelect`. If `getValue` is omitted, the whole object is the selected value.

```tsx
import { JBOptionList, JBSelect } from 'jb-select/react';

const people = [
  { id: 1, name: 'Ava Patel', role: 'Designer' },
  { id: 2, name: 'Sam Lee', role: 'Engineer' },
];

<JBSelect label="Assignee">
  <JBOptionList
    optionList={people}
    getTitle={(person) => `${person.name} — ${person.role}`}
    getValue={(person) => person.id}
  />
</JBSelect>;
```

### Custom content with `JBOptionList`

When an array-driven option needs non-text content, use `getContentDOM` and return an `HTMLElement`. If the option UI is naturally React JSX, prefer mapping to `JBOption` instead—it is simpler and keeps the rendering declarative.

```tsx
<JBOptionList
  optionList={people}
  getTitle={(person) => person.name}
  getValue={(person) => person.id}
  getContentDOM={(person) => {
    const element = document.createElement('div');
    element.textContent = `${person.name} — ${person.role}`;
    return element;
  }}
/>
```

## Using `JBOptionList` with multiple selection

The option source does not change the selection behaviour. Add `multiple` to the parent `JBSelect`; its value will contain an array of the values returned by `getValue`.

## Practical rule

Start with `JBOptionList` for simple API or data-driven lists. Switch to `JBOption` when the visible content needs React components or varies per option. Avoid nesting `JBOption` inside `JBOptionList`: choose one approach for each option collection.
